## 1. Objetivo deste documento
Este documento organiza **todos os itens obrigatórios** exigidos no roteiro do professor para a entrega do Trabalho II, garantindo que nada fique de fora.

**Regra principal:** a entrega final deve ser **um único arquivo PDF**, contendo todos os itens reunidos e organizados.

---

## 2. Itens obrigatórios (sem exceções)

### 2.1 User Stories e cartões de acompanhamento
**O que entregar:**
- User Stories do sistema (ex.: US01, US02, US03...);
- Cartões de acompanhamento de cada User Story (status, responsável, prioridade, sprint/etapa).

**Evidência no PDF:**
- Tabela com todas as User Stories;
- Imagens dos cartões (ou tabela equivalente) mostrando o acompanhamento.

---

### 2.2 Imagem do processo de desenvolvimento no Trello
**O que entregar:**
- Captura(s) de tela do Trello mostrando o fluxo de trabalho (colunas e movimentação dos cartões).

**Evidência no PDF:**
- Print legível do board;
- Breve explicação do fluxo (ex.: Backlog -> Em andamento -> Revisão -> Concluído).

---

### 2.3 Front-end completo do sistema
**O que entregar:**
- Interface completa com as telas planejadas no projeto;
- Navegação funcional entre telas.

**Evidência no PDF:**
- Prints de todas as telas principais;
- Descrição curta de cada tela e sua função.

---

### 2.4 Back-end (requisito primário + 1 requisito secundário)
**O que entregar:**
- Implementação do back-end referente ao requisito funcional primário;
- Implementação de pelo menos 1 requisito funcional secundário;
- Integração com **no mínimo 2 telas** do front-end.

**Evidência no PDF:**
- Lista dos endpoints implementados;
- Print de testes (Postman/Insomnia/cURL ou logs);
- Prints da integração funcionando nas telas.

---

### 2.5 Banco de dados integrado ao sistema
**Obrigatório para:** alunos acima do 2o semestre.

**O que entregar:**
- Banco conectado ao back-end;
- Persistência real de dados.

**Evidência no PDF:**
- Diagrama ou descrição das tabelas;
- Print da estrutura do banco;
- Print de registros persistidos.

---

### 2.6 Uso de Spring Boot
**O que entregar:**
- Aplicação backend construída com Spring Boot (standalone/produção).

**Evidência no PDF:**
- Print do `pom.xml` com dependências Spring Boot;
- Print da aplicação em execução (`mvn spring-boot:run` / log de startup).

---

### 2.7 Relatório das entrevistas (Engenheiros de Requisitos ou stakeholders)
**O que entregar:**
- Relatório das entrevistas realizadas;
- Principais perguntas e respostas;
- Requisitos levantados a partir das entrevistas.

**Evidência no PDF:**
- Seção específica chamada "Relatório de Entrevistas";
- Nome/perfil do entrevistado (quando aplicável), data e síntese dos resultados.

---

### 2.8 Repositório versionado no GitHub
**O que entregar:**
- Repositório com código-fonte e documentação do projeto.

**Evidência no PDF:**
- Link do repositório;
- Print da página principal no GitHub;
- Print de commits relevantes.

---
